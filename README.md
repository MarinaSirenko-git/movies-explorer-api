### Ключевые понятия

`Express`, `Mongo`, `Node.js`,`Yandex Cloud`, `SSL`, `Nginx`, `Winston`, `Jest`

### Коротко о проекте
Сервис, в котором можно найти фильмы по запросу и сохранить в личном кабинете. 
Работает так: пользователь проходит регистрацию, авторизацию, получает доступ к своему аккаунту и поисковую строку. 
Вводит в строку ключевые слова и нажимает кнопку «Искать». 
После этого сайт должен выполнить два действия:
отправить запрос к сервису с данными о фильмах, получить данные и сохранить;
если пользователь сохраняет фильм, он должен отображаться в специальном разделе сайта.

### Коротко о задаче
Реализовать API для аутентификации пользователей и сохранения фильмов. При этом пароль хранится в зашифрованном виде.
Данные в теле и параметрах запроса валидируются. Ошибки обрабатываются централизованно. Используются статусы ошибок 400, 401, 403, 404, 409, 500.
У пользователя нет возможности удалять фильмы других пользователей. Jwt хранится в cookie под флагом httpOnly. Роуты защищены авторизацией.

### Деплой (Render Web Service)

1. Создайте кластер [MongoDB Atlas](https://www.mongodb.com/atlas), пользователя БД и в Network Access разрешите доступ с IP Render (часто `0.0.0.0/0` для разработки или исходящие IP из [документации Render](https://render.com/docs/outbound-ip-addresses)).
2. В Render: **New → Web Service**, подключите репозиторий, **Build Command** можно оставить пустым или `npm ci`, **Start Command:** `npm start`.
3. Задайте переменные окружения:

| Переменная | Описание |
|------------|----------|
| `MONGODB_URI` | Строка подключения MongoDB (Atlas) |
| `SECRET_KEY` | Секрет для JWT (не используйте значение по умолчанию из кода) |
| `ALLOWED_ORIGINS` | Опционально: список origin фронтенда через запятую, например `https://movies-explorer-frontend.pages.dev` |
| `PORT` | На Render обычно задаётся автоматически |

4. Для проверки работоспособности: `GET /health` должен вернуть `{ "ok": true }`.

Локально скопируйте `.env.example` в `.env` и заполните значения.
